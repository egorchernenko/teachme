Здарова, тут будет дока по серваку ептиль ну а если че пиши

Для начала установи Postman и импортируй вот этот json файл.
На сылки не нажимай, ничего не произойдет, тестить только через Postman.

Сама дока:

  1. Регестрация юзера

    POST запрос /users

    body:
    {
    "name": "Pussy",
    "surname": "Golder",
    "email": "wet@mail.com",
    "password": "qwe123"
    }

    header:
    Content-Type : application/json

  2. Логин

    POST запрос /users/login

    body:
    {
    "email": "wet@mail.com",
    "password": "qwe123"
    }

    header:
    Content-Type : application/json

  3. Выход

    DELETE запрос /users/me/token

    header:
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  4. Добавление нового курсa

    POST запрос /subjects, каждый юзер может создать свой собественный курс

    body:
    {
    "name": "Guitar teacher",
    "description": "Bryan brayn tudu du",
    "schedule": [
    {"day": "Thuesday", "start": "10-00", "end": "10-45"},
    {"day": "Wednesday", "start": "11-00", "end": "11-45"},
    {"day": "Thirday", "start": "13-00", "end": "14-00"},
    {"day": "Friday", "start": "11-00", "end": "11-45"},
    {"day": "Saturday", "start": "11-00", "end": "11-45"}
    ]
    }

    header:
    Content-Type : application/json
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  5. Получение списка курсов которые ты ведешь

    GET запрос /subjects

    header:
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  6. Получение курса который ты ведешь по его ID

    GET запрос /subjects/:id

    header:
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  7. Удаление курса который ты ведешь по его ID

    DELETE запрос /subjects/:id

    header:
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  8. Обновление информации по курсу

    PATCH запрос /subjects/:id

    body:
    {
    "name": "Guitar teacher",
    "description": "Bryan brayn tudu du",
    "schedule": [
    {"day": "Thuesday", "start": "10-00", "end": "10-45"},
    {"day": "Wednesday", "start": "11-00", "end": "11-45"},
    {"day": "Thirday", "start": "13-00", "end": "14-00"},
    {"day": "Friday", "start": "11-00", "end": "11-45"},
    {"day": "Saturday", "start": "11-00", "end": "11-45"}
    ]
    }

    header:
    Content-Type : application/json
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  9. Получение всех существующих курсов, например для отображения на главной

    GET запрос /subjects/all

  10. Подписка на курс по его ID

    PATCH запрос /users/add/subject/:id

    header:
    Content-Type : application/json
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  11. Получить список курсов на которые подписан ты

    GET запрос /users/subjects/subscribed

    header:
    x-auth: сюда токен (юзер обязательно должен быть залогинен)

  • x-auth - это токен, получаем мы его при регестрации или при логине, нужен для того что бы совершать действия над юзером
